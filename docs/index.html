The pivot · v1 to v2

The v1 mix had no real flow

v1 generated the beat and the vocal with two unrelated models and stuck them together with mix.py. The output is technically a "song" — beat plus vocal — but the vocal isn't in the beat's key, isn't on the beat's grid, and doesn't know the beat exists. There is no real flow.

v2 swaps the three-model + mixer stack for one model: ACE-Step v1.5, a 3.5B-parameter diffusion model trained on lyrics + style tags → full song. Vocal and instrumental come out the same generation, musically aligned by construction. Apache 2.0 weights.

v1 stack archived

  • MusicGen-small writes the beat from a text prompt
  • Bark v2/en_speaker_6 reads the lyrics as speech-with-singing
  • mix.py resamples, pads, blends with hand-tuned gains
  • Three separate models, three separate generations, one Python script to glue
  • Vocal has no idea what key or tempo the beat is in
  • Tracks 01–03

v2 stack current

  • ACE-Step v1.5 generates vocals + instrumental together
  • Single lyric + style-tag prompt; section markers ([verse], [chorus]) control structure
  • 60-step diffusion, ~8 s of compute on an H100 slice
  • Vocal lands on the beat because the same model emitted both
  • RapMachine LoRA available for hip-hop-specific fine-tuning
  • Track 04
Lyrics · Culture to Code
[verse] Living life like a movie scene Breaking barriers, living my dream From Bangalore streets to worldwide beats Every verse I drop is pure heat [verse] Mix the culture with the flow Ancient wisdom with the modern show Breaking boundaries as I go This is how we make it grow

The [verse] markers are ACE-Step structure tags — the model uses them to plan section transitions. These lyrics are the v1 prompt that originally lived in hanumankind_song.py; the v2 engine takes them straight through with no additional formatting beyond the structure markers.

v2 engine · ACE-Step in song.py
# song.py — AIArtist v2 entry point
from acestep.pipeline_ace_step import ACEStepPipeline

pipeline = ACEStepPipeline(
    checkpoint_dir="",            # auto-pulls ACE-Step/ACE-Step-v1-3.5B from HF
    dtype="bfloat16",
    cpu_offload=True,            # for 8 GB GPUs; turn off if you have 16+ GB
    overlapped_decode=True,
)

pipeline(
    audio_duration=60,
    prompt="hip-hop, rap, Indian fusion, tabla, sitar samples, 808 bass, "
           "trap drums, 90 BPM, gritty, lyrical rap, male vocal, melodic hooks",
    lyrics="""[verse]\nLiving life like a movie scene\n...""",
    infer_step=60,
    guidance_scale=15.0,
    scheduler_type="euler",
    cfg_type="apg",
    manual_seeds="42",
    save_path="output/track.wav",
)

Full script: song.py. Track 04 was generated with these exact parameters, seed 42, against the public ACE-Step demo Space (zero-A10G) via gradio_client.

Run the v2 pipeline
# 1. install ACE-Step (separate repo, Apache 2.0)
git clone https://github.com/ace-step/ACE-Step
cd ACE-Step && pip install -e .

# 2. clone AIArtist and run song.py
cd ..
git clone https://github.com/windycityassassin/AIArtist
cd AIArtist
python song.py --duration 60 \
               --prompt "hip-hop, Indian fusion, 90 BPM, gritty" \
               --out output/track.wav

GPU recommended. 8 GB VRAM works with --cpu-offload; 12 GB is comfortable; more is faster. CPU-only inference works but takes hours per song.

About the artist

v2/en_speaker_6 is not a person. It was originally named after the Bark voice preset that produced the v1 vocals. As of v2 the voice is whatever ACE-Step generates for the given lyric and style prompt — but the name stuck. The artist is a library identifier all the way down.

The artistic target is Indian-fusion hip-hop. The reference for the style prompt is Hanumankind, an Indian rapper. The model does not impersonate; it reaches in the direction of the prompt.

About this page

The interesting layer used to live in a sibling repo. As of May 2026 the engine moved into AIArtist so the page and the source tree share a name. v1 stems are kept as the "before"; track 04 is the "after." The mix.py code excerpt is preserved on the older revision.

This is still a product mockup, not a real launch. The point is to answer one question: what does an end-to-end synthetic music release feel like when you build everything yourself, from the model down to the artwork?

What this is not

Not an attempt to pose as a real artist. The name is a library identifier, the cover art is CSS gradients, every reference says synthetic. Track 04's vocal sounds like a real performer because ACE-Step's training data sounds like real performers; that's a property of the model, not a claim about the singer.

Not a SaaS. The repo is the engine and the page is the showcase. No API key in the request path; ACE-Step weights are pulled from HuggingFace at first run and cached.

Not a voice clone of a real artist. The Hanumankind reference is a genre prompt for the beat side, not a vocal impersonation. ACE-Step generates a fresh vocal timbre per seed.
