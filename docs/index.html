Lyrics · Culture to Code
Living life like a movie scene, Breaking barriers, living my dream, From Bangalore streets to worldwide beats, Every verse I drop is pure heat. Mix the culture with the flow, Ancient wisdom with the modern show, Breaking boundaries as I go, This is how we make it grow.

Lyrics are the input passed to the TTS layer. They were written by a human and committed to aisong/hanumankind_song.py as the example prompt. The TTS model then turned that text into the vocal you hear above. Same lyrics every run; voice changes only when the preset changes.

About the artist

v2/en_speaker_6 is not a person. It is one of Bark's pretrained voice presets — the one the original repo's docs describe as best for singing. Treating it as an artist was the conceit that made the release legible: every track has a producer, a vocalist, and a cover, and now all three are software components.

The genre target is Indian-fusion hip-hop, prompted into the MusicGen beat. The reference for the prompt was Hanumankind, an Indian rapper. The model does not impersonate; it reaches in the direction of the prompt.

About this page

This is a product mockup, not a real launch. There is no streaming distribution, no royalties, no fan club, no tour. The page exists to answer one question: what does an end-to-end synthetic music release feel like when you build everything yourself, from the model down to the artwork?

The cover art is CSS gradients. The audio is real WAV. The name is a string from a Python library. The whole stack is open weights.

What this is not

Not an attempt to deceive anyone into thinking a real human made this music. The artist name is a library identifier. The cover art is gradients. The about-text says synthetic at the top. The credits link to the engine.

Not a SaaS for AI artist creation. The interesting layer (the model pipeline) is in aisong. This page is the showcase shell — what the output looks like when you put it on a release page.
