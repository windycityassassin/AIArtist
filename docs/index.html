Engine · how a track is made

Three models, one mixer, ~120 lines of code

01
prompt the beat
A single natural-language prompt describes genre, instruments, mood. Example: "hip-hop beat with Indian fusion elements, strong bass, dynamic percussion."
02
MusicGen beat
facebook/musicgen-small generates the instrumental at 32 kHz mono. guidance_scale = 3.0, do_sample = True. 30 seconds takes ~45 s on a free-tier GPU.
03
vocal synthesis
Two paths. Bark with the v2/en_speaker_6 preset (24 kHz, best for singing) is the default. XTTS v2 with a reference WAV is the voice-clone path for users supplying their own sample.
04
mix
mix.py resamples the vocal to the beat's rate, offsets it by a configurable lead-in, loops the beat to the vocal length, blends at beat=0.55, vocal=0.85, peak-normalizes to −0.5 dBFS, writes mono int16 WAV.
Code · mix.py, the part that was the TODO
# mix.py — closes the TODO that used to live at hanumankind_song.py:58
# scipy + numpy, no model deps, no GPU.

def mix(beat_path, vocal_path, out_path,
        beat_gain=0.55, vocal_gain=0.85, vocal_offset_s=0.0):
    beat_sr, beat   = load_mono(beat_path)
    vocal_sr, vocal = load_mono(vocal_path)

    # vocal -> beat rate. beat sets the master rate.
    vocal = resample_to(vocal, vocal_sr, beat_sr)
    sr    = beat_sr

    # optional silence at the front of the vocal (count-in)
    if vocal_offset_s > 0:
        pad   = np.zeros(int(vocal_offset_s * sr), dtype=vocal.dtype)
        vocal = np.concatenate([pad, vocal])

    # beat loops to the vocal length if the vocal is longer; otherwise vocal pads.
    target  = max(len(beat), len(vocal))
    beat    = pad_or_loop(beat,  target, loop=True)
    vocal   = pad_or_loop(vocal, target, loop=False)

    mixed = beat_gain * beat + vocal_gain * vocal

    # peak-normalize to -0.5 dBFS
    peak = float(np.max(np.abs(mixed)))
    if peak > 0:
        mixed = mixed * (10 ** (-0.5 / 20) / peak)

    out = np.clip(mixed * 32767.0, -32768, 32767).astype(np.int16)
    wavfile.write(out_path, sr, out)
    return out_path, sr, len(out) / sr

Steps 1 and 2 download ~3 GB of model weights on first run and want a GPU. Step 3 runs anywhere — that's the only piece that produced track 03.

About the artist

v2/en_speaker_6 is not a person. It is one of Bark's pretrained voice presets — the one the original library notes flag as best for singing. Treating it as an artist was the conceit that made the release legible: every track has a producer, a vocalist, and a cover, and now all three are software components.

The genre target on the beat side is Indian-fusion hip-hop, prompted into MusicGen. The reference for that prompt was Hanumankind, an Indian rapper. The model does not impersonate; it reaches in the direction of the prompt.

About this page

The interesting layer used to live in a sibling repo. As of May 2026 the engine moved into this repo so the page and the source tree share a name. The discography, the lyrics, the pipeline diagram, and the mixer code all sit under one URL.

This is still a product mockup, not a real launch — no streaming distribution, no royalties, no fan club. It exists to answer one question: what does an end-to-end synthetic music release feel like when you build everything yourself, from the model down to the artwork?

What this is not

Not an attempt to deceive anyone into thinking a real human made this music. The artist name is a Bark library identifier, the cover art is CSS gradients, the about-text says synthetic at the top, and the credits link out to the open-weights upstream.

Not a voice clone of a real artist. The Hanumankind reference is a genre prompt for the beat, not a vocal impersonation. The XTTS path is opt-in and requires a self-provided reference WAV.

Not a SaaS. The repo is the engine and the page is the showcase; there's no API key in the request path.
