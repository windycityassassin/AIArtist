# AIArtist

A fictional musician whose entire catalogue is generated by the [aisong](https://github.com/windycityassassin/aisong) pipeline. This repo is the product mockup for what an AI-only release looks like: an artist landing page that bundles two real aisong outputs as tracks, with credits, cover art, and liner notes.

**Live release page:** [windycityassassin.github.io/AIArtist/](https://windycityassassin.github.io/AIArtist/)

## What it is

A single static HTML page. No backend, no streaming infrastructure, no JS framework. Cover art is CSS gradients. The artist name is `v2/en_speaker_6` — the Bark voice preset that produced the vocal.

Two tracks served directly from this repo:

- `docs/audio/track_01_first_light.wav` — Bark vocal only, from `aisong/generate_song.py`
- `docs/audio/track_02_culture_to_code.wav` — MusicGen beat + Bark vocal, from `aisong/hanumankind_song.py`

## Why

To answer one question: what does an end-to-end synthetic music release feel like when you build everything yourself, from the model down to the artwork?

The interesting layer (the pipeline) is in [aisong](https://github.com/windycityassassin/aisong). This page is the showcase shell — what the output looks like when you put it on a release page.

## Stack

HTML · CSS · two WAVs from upstream `aisong`

## Not

Not an attempt to deceive anyone into thinking a real human made this music. The artist name is a library identifier, the cover art is gradients, the about-text says synthetic at the top, and the credits link to the engine.

Not a SaaS for AI artist creation.
